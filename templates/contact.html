Contact SubForge

Bug reports, feature requests, questions — we're responsive and open to contributions.

How to reach us
Bug Report GitHub Issues
Found something broken? Open an issue with steps to reproduce. Include your OS, browser, and the file type you were processing.
Feature Request GitHub Issues
Have an idea to improve SubForge? Describe the use case and the problem it solves. We prioritize requests with clear motivation.
Security Vulnerability 48h response
Please report security issues privately via GitHub Security Advisories, or open an issue with the security label. Do not disclose publicly before we respond. See our Security page for details.
General Discussion
Questions, usage tips, or community discussion — GitHub Discussions is the place. Great for self-hosting questions and deployment advice.
Response Times
Bug reports
2–5 business days
Security issues
Within 48 hours
Feature requests
Reviewed weekly
Pull requests
Reviewed weekly

SubForge is maintained as an open-source project. There is no dedicated support team — responses come from maintainers on a best-effort basis. For production deployments requiring SLA guarantees, consider self-hosting with your own team support.
