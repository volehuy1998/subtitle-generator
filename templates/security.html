Risk Level: Low
Attack Prevention

File Upload Security

Every upload is validated by extension, magic bytes (MIME), size limits (2 GB), and filename sanitization. Mismatched types are rejected. ClamAV scanning available.

Rate Limiting

Uploads: 5 req/min per IP. API: 60 req/min. Brute-force protection blocks IPs after 10 failed auth attempts for 10 minutes.

No Shell Injection

All ffmpeg/ffprobe commands use list-based argument passing (no shell=True). Font names, colors, and positions are strictly validated.

No SQL Injection

All database access uses SQLAlchemy ORM with parameterized queries. Raw SQL uses named bindings (:param). No string concatenation in queries.

Path Traversal Blocked

Upload filenames are replaced with random UUIDs. Download paths are verified to be within the output directory. No user-controlled path segments.

Task Ownership (IDOR Prevention)

Each task is bound to the session that created it via a secure httponly cookie. Cross-session access returns HTTP 403.

Authentication

Optional API key auth enforces all non-public endpoints when configured. JWT tokens with HS256 signing. Timing-safe key comparison.

Secrets Management

No hardcoded secrets. All credentials (API_KEYS, JWT_SECRET, DATABASE_URL, S3 keys) loaded exclusively from environment variables.

Data Handling & Retention

Uploaded files are temporary

Files are deleted automatically within 24 hours. They are never used for training or shared with third parties.

No account required

No personal information is collected. Sessions are anonymous, identified only by a random cookie.

Access logs retained for security

Request logs (IP, endpoint, timestamp) are kept for abuse detection. Logs do not contain file contents or transcript text.

No third-party data sharing

Transcription runs entirely on-server using open-source models (faster-whisper / CTranslate2). Nothing leaves the server.
